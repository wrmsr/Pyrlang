term.pid module¶

class term.pid.Pid(node_name: str, id: int, serial: int, creation: int) → None¶

Bases: term.bases.BasePid

Represents Erlang-style process identifier with 3 components. Node component is always 0 for local node, otherwise it can take arbitrary integer values.

A pid is created by Erlang automatically. Pyrlang Process also generates one when it is born to register itself in the process registry. Pid uniquely identifies a running process in the cluster.
